13.2 The Big O of the linear search is O(n), and the Big O of the binary search is O(log n). What key aspect of both the binary search (Chapter 6) and the merge sort accounts for the logarithmic portion of their respective Big Os?

book answer
Both algorithms incorporate “halving”, somehow reducing something by half on each pass.
The binary search eliminates from consideration one-half of the array after each comparison.
The merge sort splits the array in half each time it’s called.

from Gemini
Both utilize a divide-and-conquer strategy.
The logarithmic time complexity (\(\log n\)) in both binary search and merge sort is derived from the divide-and-conquer strategy, which repeatedly divides the dataset in half during each step.
Binary search halves the search space each time, while merge sort reduces the array into sub-arrays of size one, with both algorithms taking \(\log _{2}n\) steps to reach a base case.

13.3 In what sense is the insertion sort superior to the merge sort? In what sense is the merge sort superior to the insertion sort?

book answer
The insertion sort is easier to understand and implement than the merge sort.
The merge sort is far more efficient—O(n log n)—than the insertion sort—O(n2).
