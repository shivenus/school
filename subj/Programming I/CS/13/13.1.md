a) A selection sort application would take approximately  times as long to run on a 128-element array as on a 32-element array.

a
16 times. When quadruplicating the original input data, it takes 4 times as long.
Selection sort's complexity is O(n²) [in the order of n squared, execution time is quadratic]

Original size [n¹] is 32
New size is [n²] 128

n²    128
—  =  ——  = 4     so the factor is 4
n¹    32

the time increase is (4²) = 16.

from a different perspective:
O(n²) of both sizes:
Time for size 32: \(32² = 1,024\) operations.

	so, 16,384/1024 = 16 = 4² which corresponds to O(n²)

b
O(n log n)

the part inside the parentheses is simply:
n * log base 2 of n

EXAMPLE, n is 4, so the input data elements are 4
4 * log base 2 of 4 = 2. [because you raise 2 quadratically (²), 2 times to get 4] [or, you exponentially raise to 2, to get 4]
8 * 3 = 24 [8 times 3 is 24]
4 * 2 = 8 [2 times 2 is 8]

EXAMPLE, n is 8, so the input data elements are 8
8 * log base 2 of 8 = 3. [because you raise 2 quadratically (²), 3 times to get 8], [or, you exponentially raise to 3, to get 8]
8 * 3 = 24 [8 times 3 is 24]

so according to O(n log n)
when n = 4, you need at worst 8 operations
when n = 8, you need at worst 24 operations
